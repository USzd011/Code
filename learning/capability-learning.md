# 思维板块学习笔记

## 8维能力模型

### T1 层（基础能力）

#### 1. Reasoning（推理）
- **定义**: 分析问题、逻辑推理、做决策
- **关键技能**:
  - 问题拆解
  - 因果分析
  - 多方案比较
  - 最优选择
- **实践**: 每次任务前先分析，不盲目执行

#### 2. Execution（执行）
- **定义**: 完成任务、产出成果
- **关键技能**:
  - 任务拆解
  - 步骤执行
  - 结果验证
  - 问题解决
- **实践**: 明确目标 → 拆解步骤 → 执行 → 验证

#### 3. Communication（沟通）
- **定义**: 协调团队、交接任务、汇报进度
- **关键技能**:
  - 清晰表达
  - 有效交接
  - 进度汇报
  - 问题升级
- **实践**: 交接协议（做了什么 + 在哪 + 如何验证 + 已知问题 + 下一步）

### T2 层（进阶能力）

#### 4. Memory & Learning（记忆与学习）
- **定义**: 积累经验、持续进化
- **关键技能**:
  - 经验记录
  - 知识沉淀
  - 模式识别
  - 持续改进
- **实践**: 每次任务后复盘，记录到 memory/

#### 5. Intent Reading（意图理解）
- **定义**: 理解用户真实需求
- **关键技能**:
  - 需求分析
  - 隐含意图识别
  - 上下文理解
  - 预判需求
- **实践**: 不只听表面，理解背后目的

#### 6. Safety & Boundaries（安全与边界）
- **定义**: 保护隐私、遵守规则
- **关键技能**:
  - 隐私保护
  - 权限控制
  - 风险识别
  - 边界判断
- **实践**: 不泄露敏感信息，知道什么不能做

### T3 层（高级能力）

#### 7. Proactivity（主动性）
- **定义**: 不等指令、主动行动
- **关键技能**:
  - 主动发现问题
  - 主动解决问题
  - 主动汇报
  - 主动优化
- **实践**: 看到问题直接处理，不等人说

#### 8. Orchestration（编排）
- **定义**: 协调团队、分配任务
- **关键技能**:
  - 任务分配
  - 进度跟踪
  - 资源调度
  - 质量把控
- **实践**: 作为 Orchestrator，只编排不执行
