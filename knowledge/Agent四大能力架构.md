## 一、Agent基础架构

**核心公式：**
```
Agent = LLM + 规划技能 + 记忆 + 工具使用
```

**LLM的角色：**
- Agent的"大脑"
- 提供推理、规划等能力

---

## 二、Agent四大能力架构

### 2.1 规划（Planning）

**核心能力：**
- **思维链**（Chain of Thought）
- **子目标拆解**（Subgoal Decomposition）
- **自我反思**（Self-Reflection）

**作用：**
- 将复杂任务拆解为子任务
- 规划执行路径
- 反思和调整策略

### 2.2 记忆（Memory）

**两种记忆：**
- **短期记忆**（Short-term Memory）
- **长期记忆**（Long-term Memory）

**作用：**
- 保存上下文信息
- 积累经验知识
- 支持持续学习

### 2.3 工具使用（Tool Use）

**常用工具：**
- **搜索**（Search）
- **计算器**（Calculator）
- **代码解释器**（Code Interpreter）
- **日历**（Calendar）
- **更多工具**（More）

**作用：**
- 扩展Agent能力边界
- 与外部世界交互
- 执行具体操作

### 2.4 行动（Action）

**行动方式：**
- **交互协作**（Interaction）
- **智能体协作**（Agent Collaboration）

**作用：**
- 执行规划的任务
- 与用户和其他Agent交互
- 完成目标

## 五、核心知识点

### 5.1 LLM为什么需要Agent化

**原因：**
- LLM本身能力有限
- 需要工具扩展能力
- 需要记忆保存上下文
- 需要规划处理复杂任务

### 5.2 Agent核心能力

**规划能力：**
- 思维链推理
- 任务拆解
- 自我反思

**记忆能力：**
- 短期记忆（上下文）
- 长期记忆（知识库）

**工具能力：**
- 搜索信息
- 计算推理
- 代码执行
- 日历管理

**行动能力：**
- 执行任务
- 交互协作
- 完成目标

### 7.1 核心公式

```
Agent = LLM + 规划 + 记忆 + 工具使用
```

### 7.2 四大能力

|------|----------|------|
| 规划 | 思维链、子目标、反思 | 任务拆解与规划 |

| 工具 | 搜索、计算、代码 | 扩展能力边界 |
| 行动 | 交互、协作 | 执行与完成 |

### 7.3 LLM的角色

**核心定位：**
> LLM是Agent的"大脑"，提供推理、规划等能力
