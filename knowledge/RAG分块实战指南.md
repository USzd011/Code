
## 三、分块的本质

### 3.1 分块的必要性

**源于两个核心限制：**

**限制1：模型上下文窗口**
```
大语言模型(LLM)无法一次性处理无限长度的文本
分块是将长文档切分为模型可以处理的、大小适中的片段
```

**限制2：检索信噪比**
```
如果一个文本块包含过多无关信息（噪声），就会稀释核心信号
导致检索器难以精确匹配用户意图
```

### 3.2 理想的分块

**核心目标：**
```
在上下文完整性与信息密度之间找到最佳平衡
```

**关键参数：**
- **chunk_size** - 分块大小
- **chunk_overlap** - 分块重叠

**chunk_overlap的作用：**
```
通过在相邻块之间保留部分重复文本
确保了跨越块边界的信息不会丢失
```

## 四、分块策略分类

### 4.1 基础分块策略

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| **固定长度分块** | 简单直接，按字符数切分 | 格式统一的文档 |
| **递归字符分块** | 按分隔符层级递归切分 | 结构化文档 |
| **基于句子分块** | 按句子边界切分 | 需要语义完整的场景 |

### 4.2 结构感知分块

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| **结构化文档分块** | 按文档结构（标题、段落）切分 | Markdown、HTML等 |
| **对话式分块** | 按对话轮次切分 | 聊天记录、客服对话 |

### 4.3 语义与主题分块

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| **语义分块** | 基于语义相似度切分 | 需要语义连贯的场景 |
| **基于主题分块** | 按主题变化切分 | 多主题文档 |

### 4.4 高级分块策略

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| **小大分块** | 小块检索，大块回答 | 需要精确检索+完整上下文 |
| **代理式分块** | AI代理动态决定分块 | 复杂文档处理 |

## 五、实战要点

### 5.1 分块策略选择

**选择原则：**
```
文档特点 + 检索需求 + 模型能力 → 选择合适策略
```

**选择建议：**

| 文档类型 | 推荐策略 |
|----------|----------|
| 技术文档 | 递归字符分块 + 结构感知 |
| 对话记录 | 对话式分块 |
| 长文章 | 语义分块 |
| 多主题文档 | 基于主题分块 |
| 复杂文档 | 代理式分块 |

### 5.2 参数调优

**chunk_size选择：**
- 太小：上下文不完整
- 太大：信噪比降低
- 建议：根据模型窗口和文档特点调整

**chunk_overlap选择：**
- 确保跨块信息不丢失
- 建议：chunk_size的10-20%
