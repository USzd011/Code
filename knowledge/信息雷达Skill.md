> 来源：小红书 - 瑶瑶AI增长

---

## 一、Skill概述

**名称：** video-info-radar
**功能：** 信息雷达
**触发词：** "抓选题"、"找选题"

---

## 二、核心功能

### 2.1 主要任务

**自动抓取外网信息源：**
```
HN、Reddit、GitHub Trending、ProductHunt等
```

### 2.2 核心逻辑

**筛选逻辑：**
```
"外网已火 + 国内没人讲" = 信息差
```

### 2.3 输出内容

**输出：**
```
按信息差排序的选题清单
├── 必拍
├── 可拍
└── 观察
```

---

## 三、信息源

### 3.1 信息源列表

**覆盖平台：**

| 平台 | 说明 |
|------|------|
| **HN** | Hacker News |
| **Reddit** | 热门讨论 |
| **GitHub** | Trending项目 |
| **ProductHunt** | 新产品 |
| **X** | Twitter热点 |

### 3.2 抓取策略

**策略：**
- 多源并行抓取
- 自动去重
- 按热度排序

---

## 四、核心逻辑

### 4.1 信息差验证

**验证流程：**
```
外网热度 → 自动搜中文互联网 → 验证信息差程度 → 排序输出
```

### 4.2 筛选标准

**筛选逻辑：**
```
外网已火 + 国内没人讲 = 高信息差 = 必拍
```

---

## 五、输出分类

### 5.1 三类选题

**必拍：**
```
信息差高 + 热度高 + 时效性强
建议24小时内出视频
```

**可拍：**
```
信息差中 + 热度中 + 有价值
可以安排拍摄
```

**观察：**
```
信息差低或热度低
持续观察，等待时机
```

### 6.1 效率价值

**价值：**
- 自动抓取多源信息
- 一键完成全网扫描
- 大幅提升选题效率

### 6.2 信息差价值

**价值：**
- 自动验证信息差
- 发现蓝海选题

### 6.3 时效价值

**价值：**
- 实时抓取热点
- 快速响应
- 抢占先机
