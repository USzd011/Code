# 演示规划Skill - video-demo-planner

> 来源：小红书 - 瑶瑶AI增长
> 整理时间：2026-05-04

---

## 一、Skill概述

**名称：** video-demo-planner
**功能：** 规划演示

**核心目标：**
```
这个视频怎么演示触发
```

---

## 二、核心功能

### 2.1 主要任务

**根据脚本设计可执行的演示方案，规划"哇"时刻：**
```
每条视频必须有一个让观众震撼的画面
```

### 2.2 输出内容

**输出：**
```
演示流程 + 技术方案 + 录制计划
```

---

## 三、三种演示类型

### Type A：做工具/产品（HTML应用）

**特点：**
- 制作HTML应用
- 可视化展示
- 交互式演示

**适用场景：**
- 工具介绍
- 产品演示
- 功能展示

### Type B：操作流程录屏

**特点：**
- 录制操作过程
- 步骤清晰
- 流程展示

**适用场景：**
- 教程类视频
- 操作演示
- 流程讲解

### Type C：多场景对比展示

**特点：**
- 多个场景对比
- 效果对比
- 差异展示

**适用场景：**
- 对比评测
- 效果展示
- 案例分析

---

## 四、核心原则

### 4.1 为什么是4个独立Skill

**原因：**
```
因为有的它需要我具体的演示
```

### 4.2 设计理念

**四大原则：**
1. 每个环节可独立使用
2. 环节之间人可以介入做判断和调整
3. 单个Skill更小、更容易迭代优化
4. 不套模板，每条视频从信息差出发

---

## 五、演示规划流程

### 5.1 完整流程

```
分析脚本 → 确定演示类型 → 设计"哇"时刻 → 制定技术方案 → 规划录制计划
```

### 5.2 关键步骤

**步骤1：分析脚本**
```
理解视频内容
确定演示需求
```

**步骤2：确定演示类型**
```
Type A：HTML应用
Type B：操作录屏
Type C：多场景对比
```

**步骤3：设计"哇"时刻**
```
每条视频必须有一个让观众震撼的画面
```

**步骤4：制定技术方案**
```
确定技术实现方式
准备所需工具
```

**步骤5：规划录制计划**
```
确定录制流程
安排录制时间
```
