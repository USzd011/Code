## 一、核心公式

**Agent定义：**
```
Agent = LLM(大脑) + 规划(Planning) + 记忆(Memory) + 工具使用(Tool Use)
```

---

## 二、完整工作流程（17步）

### 阶段一：输入与规划（步骤1-3）

**步骤1：输入提示词**
```
用户 → 智能体(Agent)
"帮我分析下周销量"
```

**步骤2：任务规划（推理）**
```
智能体(Agent) → 大模型(LLM)
步骤1：查询数据库
步骤2：生成图表
```

**步骤3：规划确认**
```
大模型(LLM) → 智能体(Agent)
确认执行步骤
```

---

### 阶段二：数据库查询（步骤4-8）

**步骤4：转发标准化请求**
```
智能体(Agent) → MCP
标准化请求格式
```

**步骤5：调用数据库查询**
```
MCP → 工具
调用数据库查询工具
```

**步骤6：执行SQL查询**
```
工具执行SQL查询
```

**步骤7：返回标准化结果**
```
工具 → MCP
返回标准化格式结果
```

**步骤8：返回查询结果**
```
MCP → 智能体(Agent)
返回查询结果
```

---

### 阶段三：数据推理（步骤9）

**步骤9：数据结合推理**
```
智能体(Agent) → 大模型(LLM)
结合查询结果进行推理
```

---

### 阶段四：图表生成（步骤10-14）

**步骤10：调用图表生成工具**
```
智能体(Agent) → MCP
调用图表生成工具
```

**步骤11：转发标准化请求**
```
MCP → 工具
标准化请求格式
```

**步骤12：执行代码绘制图表**
```
工具执行代码绘制图表
```

**步骤13：返回标准化结果**
```
工具 → MCP
返回标准化格式结果
```

**步骤14：返回图表**
```
MCP → 智能体(Agent)
返回生成的图表
```

---

### 阶段五：整合与输出（步骤15-17）

**步骤15：整合数据与图表**
```
智能体(Agent) → 大模型(LLM)
整合查询结果和图表
```

**步骤16：生成最终报告**
```
大模型(LLM) → 智能体(Agent)
生成完整分析报告
```

**步骤17：返回完整分析结果**
```
智能体(Agent) → 用户
返回完整分析结果
```

## 三、四大核心组件

### 3.1 组件角色

| 组件 | 角色 | 职责 |
|------|------|------|
| **用户** | 发起者 | 输入提示词，接收结果 |
| **智能体(Agent)** | 决策中心 | 规划、调度、整合 |
| **大模型(LLM)** | 大脑 | 推理、生成、分析 |
| **MCP** | 连接标准 | 标准化请求/响应 |
| **工具** | 执行者 | 执行具体操作 |

### 3.2 协作关系

```
用户 ←→ 智能体(Agent) ←→ 大模型(LLM)
              ↓
            MCP
              ↓
           工具
```

---

## 四、工作流程特点

### 4.1 核心特点

**1. 标准化**
```
MCP协议确保请求和响应格式统一
```

**2. 模块化**
```
各组件职责清晰，可独立升级
```

**3. 可扩展**
```
工具可按需添加，能力可扩展
```

### 4.2 执行模式

**循环模式：**
```
规划 → 执行 → 推理 → 执行 → 整合 → 输出
```

**关键点：**
- 每个步骤都有明确输入输出
- MCP确保标准化交互
- 智能体负责整体调度
