# AI智能系统核心架构 - 五层协同链路

> 来源：飞书文档
> 整理时间：2026-05-04

---

## 一、核心架构概览

**五层协同链路：**
```
提示词 → 大模型 → 智能体 → MCP协议 → 工具
```

**核心价值：**
> 让AI既能"思考"，也能"行动"

---

## 二、五层架构详解

### 第1层：提示词 - 指令与控制

**角色：**
- 用户与大模型/智能体之间的"沟通语言"
- "操作指令"

**核心功能：**
- 引导大模型执行特定任务
- 扮演特定角色
- 调用特定工具

**关系：**
```
提示词是激活和驱动大模型的核心输入
```

**示例：**
```
你现在是一个旅行规划专家，请使用网络搜索工具，
为我查找上海未来三天的天气，并推荐室内外活动。
```

---

### 第2层：智能体 - 决策与执行中心

**角色：**
- 以大模型为核心的软件实体
- 增加了自主性和执行力

**核心能力：**

**1. 规划**
```
将复杂目标拆解为步骤
例如：先查天气，再查航班，最后推荐景点
```

**2. 记忆**
```
保留对话和历史操作的上下文
```

**3. 工具调用**
```
根据规划，决定在何时调用何种工具
```

**关系：**
```
智能体将大模型的通用能力与专用工具结合起来，
形成一个可以主动完成任务的"数字员工"
```

**代表产品：**
- AutoGPT
- ChatGPT的Advanced Data Analysis模式
- 各种AI助手

---

### 第3层：大模型 - 核心引擎

**角色：**
- 整个体系的"大脑"
- 基础能力提供者

**核心能力：**
- 语言理解
- 语言生成
- 逻辑推理

**类比：**
```
就像是人类的大脑，能够思考、分析和创造
```

**代表模型：**
- DeepSeek
- Qwen
- GPT-4
- Claude
- Llama

---

### 第4层：MCP模型上下文协议 - 连接标准

**角色：**
- 智能体与现实世界交互的接口
- 获取实时信息或执行具体操作的插件

**核心价值：**
大模型本身是纯思维的，它不知道实时天气，不能执行代码，也不能操作数据库，工具弥补了这一缺陷。

**三大特性：**

**1. 标准化**
```
工具开发者按照统一的MCP标准来开发工具
```

**2. 安全性**
```
明确定义了工具的可访问资源和操作权限
```

**3. 互操作性**
```
任何支持MCP的智能体都可以轻松使用任何工具
```

**关系：**
```
智能体通过调用工具来扩展其能力边界，从思考到行动
```

**工具示例：**
- **网络搜索工具** - 获取最新信息
- **代码执行器** - 运行代码并返回结果
- **API调用工具** - 与其他软件服务交互

## 三、完整链路

### 3.1 五层协同

```
┌─────────────┐
│   提示词    │ → 激发意图
└──────┬──────┘
       ↓
┌─────────────┐
│   大模型    │ → 提供认知
└──────┬──────┘
       ↓
┌─────────────┐
│   智能体    │ → 决策与调度
└──────┬──────┘
       ↓
┌─────────────┐
│  MCP协议    │ → 标准化连接
└──────┬──────┘
       ↓
┌─────────────┐
│    工具     │ → 执行现实操作
└─────────────┘
```

### 3.2 核心价值

**协同链路：**
```
提示词激发意图 → 大模型提供认知 → 智能体负责决策与调度 → MCP标准化连接 → 工具执行现实操作
```

**最终目标：**
> 五者协同构成智能系统的完整链路，让AI既能"思考"，也能"行动"

## 四、MCP协议详解

### 4.1 为什么需要MCP

**问题：**
- 大模型是纯思维的
- 不知道实时天气
- 不能执行代码
- 不能操作数据库

**解决：**
- MCP协议提供标准化连接
- 工具弥补能力缺陷

### 4.2 MCP三大价值

| 价值 | 说明 |
|------|------|
| **标准化** | 统一开发标准 |
| **安全性** | 明确权限边界 |
| **互操作性** | 跨平台使用 |

### 4.3 工具类型

**常用工具：**
- 网络搜索工具
- 代码执行器
- API调用工具
- 数据库操作工具
- 文件系统工具

## 五、架构对比

### 5.1 传统架构 vs MCP架构

| 维度 | 传统架构 | MCP架构 |
|------|----------|---------|
| 连接方式 | 定制开发 | 标准化协议 |

| 安全性 | 自行实现 | 协议保障 |

### 5.2 MCP架构优势

**核心优势：**
1. 降低开发成本
2. 提升安全性
3. 增强互操作性
4. 简化集成流程
