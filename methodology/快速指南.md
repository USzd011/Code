## 三、工作流程

### 3.1 标准流程

```
第1步：编写PRD
├── 清晰定义目标
└── 明确验收标准

第2步：转化为用户故事
├── 分解原子任务
└── 每个任务可测试

第3步：启动循环
├── AI自主认领任务
└── Code → Test → Learn

第4步：管理记忆
├── progress.txt（短期）
└── agents.md（长期）
```

### 3.2 关键成功因素

**因素1：前期规范质量**
```
瓶颈不在编码，在前期规范
PRD质量 → 最终结果质量
```

**因素2：任务原子化**
```
每个任务必须：
├── 单一目标
├── 清晰验收标准
└── 独立可完成
```

**因素3：拥抱失败**
```
默认第一次会写错
失败 = 最有价值的反馈数据
```

## 六、立即行动

**今天就开始：**

1. 准备项目环境
   ```bash
   ./ai_dev_method.sh init my_project
   ```

2. 编写清晰PRD
   ```markdown

   ## 目标: [清晰定义]
   ```

   ```markdown
   2. 任务2（可测试）
   ```

4. 启动拉尔夫循环
   ```bash
   ./ai_dev_method.sh loop PROMPT.md
   ```

5. 收集数据优化
