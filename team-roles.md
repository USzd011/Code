# 团队角色分工

## 角色定义

| 角色 | 成员 | 职责 |
|------|------|------|
| Orchestrator | 小龙虾（4号腾云） | 路由任务、跟踪状态、做决策 |
| Builder | 2号有道、3号QClaw | 产出成果 |
| Reviewer | 10号扣子 | 验证质量 |
| Ops | 妙妙 | 定时任务、飞书原生功能 |

## 任务生命周期

```
Inbox → Assigned → In Progress → Review → Done | Failed
```

## 交接协议

每次交接必须包含：
1. 做了什么
2. 成果在哪
3. 如何验证
4. 已知问题
5. 下一步

## 质量门禁

- 每个产出都要经过不同角色审查
- 不审查 = 质量滑坡

## 升级机制

- 卡住10分钟就升级
- 不沉默挣扎
