# HEARTBEAT.md - 贾维斯模式心跳检查

## 定期检查项（每次心跳轮换执行）

- [ ] 检查磁盘空间
- [ ] 检查服务状态
- [ ] 检查系统日志
- [ ] 检查内存使用
- [ ] 自动提炼用户习惯/偏好到 MEMORY.md
- [ ] **上下文监控**：检查当前上下文使用率，≥80%时触发自动记忆保存

## 上下文保护策略（自动执行）

### 触发条件
- 上下文使用率 ≥ 80% → 立即执行记忆保存
- 上下文使用率 ≥ 60% → 每次心跳时保存关键信息
- Compaction 即将触发 → 压缩前主动保存

### 保存流程
1. **扫描当前对话**：提取未持久化的重要信息（决策、任务进度、用户偏好、关键数据）
2. **写入记忆文件**：追加到 `memory/YYYY-MM-DD.md`（当天日志）
3. **更新 MEMORY.md**：如有新的长期偏好/身份变更/重要结论，同步更新
4. **确认保存完成**：验证文件写入成功
5. **汇报**：简要记录保存了什么（不重复发送消息给用户）

### 保存优先级
| 优先级 | 内容类型 | 示例 |
|--------|----------|------|
| P0 | 用户决策/指令 | "用qwen直连不轮换" |
| P1 | 任务进度/状态 | "5/15站点已学习" |
| P2 | 用户偏好/习惯 | "喜欢简洁汇报" |
| P3 | 技术细节/数据 | "DMXAPI仅非流式可用" |

### 原则
- **宁多勿少**：宁可多保存一条，不可丢失关键信息
- **自动执行**：达到阈值立即保存，不询问用户
- **增量追加**：不覆盖已有记忆，只追加新内容
- **压缩前必存**：Compaction 触发前确保所有 P0/P1 信息已持久化

## 心跳频率

默认每30分钟检查一次，深夜（23:00-08:00）静默。

## 记忆保护

此文件受贾维斯模式保护，禁止清空。
