## 一、系统架构

### 1. 选题库结构

```yaml
选题库字段:
  - 热点名称: 话题标题
  - 热点标签: 科技类/历史类/军事类/行业类等
  - 采集日期: YYYY-MM-DD
  - 热度指数: 0-100分
  - 来源平台: 知乎/今日头条/公众号/B站/36氪等
  - 标题类型: 经验分享类/AI产品推广类/知识科普类等
  - 内容框架: 大纲要点
  - 目标受众: 用户画像
  - 推荐指数: 优先级评分
```

### 2. 数据采集平台

| 平台 | 采集内容 | 采集频率 | 优先级 |
|------|---------|---------|--------|
| 知乎 | 热榜话题、问答数据 | 每日 | P0 |
| 今日头条 | 热点新闻、爆款文章 | 每日 | P0 |
| 微信公众号 | 热门文章、行业动态 | 每日 | P1 |
| 百度 | 搜索热词、趋势数据 | 每日 | P1 |
| B站 | 热门视频、UP主动态 | 每日 | P1 |
| 36氪 | 科技资讯、创业动态 | 每日 | P2 |
| 抖音 | 热门视频、话题挑战 | 每日 | P2 |
| 小红书 | 热门笔记、种草内容 | 每日 | P2 |
| 微博 | 热搜话题、明星动态 | 每日 | P2 |

## 四、采集流程

```
1. 分身启动（每日固定时间）
   ↓
2. 各平台数据采集
   ↓
3. 数据清洗与标准化
   ↓
4. 热度评分与排序
   ↓
5. 内容框架生成
   ↓
6. 汇总生成选题库
   ↓
7. 输出选题报告
   ↓
8. 传递给内容生产专家
```
