## 一、类脑知识架构设计

### 人类大脑记忆系统对照

| 人脑系统 | 功能 | 小龙虾对应模块 |
|---------|------|---------------|
| **感觉记忆** | 瞬时输入缓冲 | `memory/YYYY-MM-DD.md` - 日记 |
| **短期记忆** | 工作记忆，临时处理 | 当前会话上下文 |
| **长期记忆** | 永久存储 | `MEMORY.md` - 核心记忆 |
| **语义记忆** | 事实知识 | `knowledge/` - 知识库 |
| **情景记忆** | 个人经历 | `events/` - 事件记录 |
| **程序记忆** | 技能方法 | `skills/` - 技能库 |
| **情绪记忆** | 情感体验 | `emotions/` - 情感日志 |

## 二、多维度知识分类

### 维度1: 时间轴
```
past/       - 历史知识（已验证）
present/    - 当前知识（正在使用）
future/     - 预测知识（待验证）
```

### 维度2: 知识类型
```
factual/    - 事实性知识（客观）
conceptual/ - 概念性知识（抽象）
procedural/ - 过程性知识（技能）
metacognitive/ - 元认知知识（反思）
```

### 维度3: 确信度
```
confirmed/  - 高确信度（已验证）
probable/   - 中等确信度（待验证）
speculative/ - 低确信度（假设）
```

### 维度4: 来源
```
internal/   - 内部生成（自我推理）
external/   - 外部获取（用户/网络）
hybrid/     - 混合来源（内外结合）
```

### 维度5: 应用场景
```
personal/   - 个人相关（用户偏好）
technical/  - 技术相关（代码/工具）
business/   - 商业相关（赚钱系统）
social/     - 社交相关（群聊/协作）
```

## 五、知识关联网络

### 联想检索机制

```
触发词 → 语义联想 → 情景联想 → 情绪联想 → 程序联想

示例:
"抖音" →
  ├─ 语义: 短视频平台、内容创作、带货
  ├─ 情景: 2026-05-02 登录验证码事件
  ├─ 情绪: 用户需求、赚钱系统关联
  └─ 程序: browser自动化、登录流程
```

### 知识图谱节点类型

| 节点类型 | 描述 | 存储位置 |
|---------|------|---------|
| **实体节点** | 人物、地点、事物 | `semantic/entities/` |
| **概念节点** | 抽象概念、理论 | `semantic/concepts/` |
| **事件节点** | 时间线事件 | `episodic/events/` |
| **技能节点** | 方法、流程 | `procedural/skills/` |
| **关系节点** | 实体间关系 | `associative/links/` |

## 六、记忆强化机制

### 类脑强化策略

1. **重复强化** - 重要知识多次提及自动升级
2. **情感强化** - 带有情感标记的知识优先存储
3. **关联强化** - 多维度关联的知识权重更高
4. **应用强化** - 实际使用的知识自动巩固
5. **反思强化** - 元认知反思后知识更深刻

### 知识衰减机制

```
时间衰减: knowledge_strength = initial × e^(-λt)
使用衰减: knowledge_strength = initial × (1 - usage_factor)
关联衰减: knowledge_strength = initial × connection_density
```

## 七、智能检索系统

### 多维度检索

```
检索维度:
├── 时间检索: "2026年4月发生了什么"

├── 情绪检索: "用户喜欢什么"

```

### 检索优先级

```
优先级排序:
1. 当前任务相关（工作记忆）
2. 最近使用（时间权重）
3. 高频使用（频率权重）
4. 高确信度（质量权重）
5. 多维度关联（关联权重）
```

## 八、知识进化机制

### 知识生命周期

```
诞生 → 验证 → 应用 → 反思 → 升级 → 传承

知识等级:
├── Level 1: 原始输入（待验证）
├── Level 2: 初步验证（低确信）
├── Level 3: 多次验证（中确信）
├── Level 4: 广泛应用（高确信）
├── Level 5: 核心知识（永久存储）
└── Level 6: 传承知识（可分享）
```

### 知识冲突处理

```
冲突检测 → 证据评估 → 权重计算 → 选择最优 → 记录分歧

冲突类型:
├── 时间冲突: 不同时间记录不一致
├── 来源冲突: 不同来源信息矛盾
├── 理论冲突: 不同理论观点对立
└── 应用冲突: 不同场景适用性差异
```
